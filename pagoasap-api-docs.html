Introducción

API de Tarjetas PagoASAP

PagoASAP es la infraestructura financiera de Venezuela que permite a fintechs y empresas no-financieras lanzar tarjetas cobranded 100% bancarias, gestionar pagos multimoneda y ofrecer payouts instantáneos — todo a través de una sola API.

https://api.pagoasap.com/v1
⚡
Diferencial clave: A diferencia de otras plataformas, PagoASAP requiere que el partner apruebe explícitamente cada transacción a través del endpoint /transactions/{id}/approve. El débito se realiza de tu balance prefunded. Tienes hasta 2 segundos para responder, garantizando control total sobre tus usuarios.
🏦
Respaldo bancario 100%: Todas las tarjetas se emiten con un banco regulado en Venezuela. PagoASAP gestiona el ciclo completo: emisión, personalización, impresión y delivery de tarjetas físicas a domicilio.

Flujo de Autorización de Transacciones

Cuando un tarjetahabiente intenta realizar una compra, PagoASAP consulta a tu sistema en tiempo real antes de autorizar:

→
⏱

Autenticación

Autenticación

PagoASAP utiliza Bearer tokens con OAuth 2.0 (Client Credentials). Incluye el token en el header Authorization de cada request.

POST /auth/token
Obtener access token (Client Credentials)
›
Body Parameters
client_id string REQUIRED
Tu Client ID de PagoASAP. Obtenido en el dashboard.
client_secret string REQUIRED
Tu Client Secret. Nunca lo expongas en el frontend.
grant_type string REQUIRED
Siempre: client_credentials
scope string
Permisos requeridos. Ej: cards:write transactions:approve
Request
Response
// POST /auth/token // Content-Type: application/x-www-form-urlencoded client_id=pk_live_abc123xyz client_secret=sk_live_secretsecret grant_type=client_credentials scope=cards:write transactions:approve // Response 200 { "access_token": "eyJhbGciOiJSUzI1NiJ9...", "token_type": "Bearer", "expires_in": 3600, "scope": "cards:write transactions:approve" }
⚠️
Los tokens de acceso expiran en 1 hora. Implementa refresco automático en tu integración. En sandbox usa el prefijo sk_test_, en producción sk_live_.

EstadoDescripción

Transacciones — Diferencial PagoASAP

Aprobar Transacción

Este es el endpoint central del modelo de PagoASAP. Cuando un tarjetahabiente intenta pagar, recibes un webhook transaction.pending y debes responder aprobando o rechazando en menos de 2 segundos. El monto se debita de tu balance prefunded.

💡
Ventaja para el partner: Puedes implementar tu propia lógica de riesgo, límites personalizados, o restricciones por categoría de merchant (MCC) antes de aprobar. Tienes control total sobre cada transacción de tus usuarios.
POST /transactions/{transaction_id}/approve
Aprobar una transacción pendiente — debita del balance prefunded
›
Path Parameters
transaction_idstringREQUIRED
ID de la transacción pendiente recibida en el webhook.
Body Parameters
reasonstring
Razón de aprobación para auditoría interna. Opcional.
metadataobject
Datos adicionales de tu sistema para correlación interna.
⏱ Timeout: Si no respondes en 2 segundos, la transacción es rechazada automáticamente y el tarjetahabiente recibe una notificación.
REQUEST
// POST /transactions/tx_8Yw3nBvRkP2m/approve { "reason": "within_daily_limit", "metadata": { "internal_ref": "ORD-20240408-009" } } // Response 200 { "id": "tx_8Yw3nBvRkP2m", "object": "transaction", "status": "approved", "amount": 45.00, "currency": "USD", "merchant": { "name": "Supermercado Central", "mcc": "5411", "city": "Caracas" }, "prefunded_balance_after": 4955.00, "approved_at": "2026-04-08T16:42:01Z" }
POST /transactions/{transaction_id}/decline
Rechazar una transacción pendiente
›
Body Parameters
decline_reasonenumREQUIRED
Código de razón: insufficient_funds, spending_limit_exceeded, merchant_blocked, fraud_suspected, partner_rule
messagestring
Mensaje descriptivo. Se mostrará al tarjetahabiente si aplica.
REQUEST
{ "decline_reason": "spending_limit_exceeded", "message": "Has superado tu límite mensual" } // Response 200 { "id": "tx_8Yw3nBvRkP2m", "status": "declined", "decline_reason": "spending_limit_exceeded", "declined_at": "2026-04-08T16:42:01Z" }

Estados de Transacción

EstadoDescripción
pendingEsperando aprobación del partner. Ventana: 2 segundos.
approvedAprobada y procesada. Balance prefunded debitado.
declinedRechazada por el partner o por timeout.
reversedAprobada pero revertida posteriormente (disputa/devolución).

El partner debe mantener un balance prefunded en PagoASAP. Este balance se debita automáticamente cuando apruebas transacciones. Configura alertas de bajo balance a través de webhooks.

Webhooks

Eventos Webhook

PagoASAP envía eventos en tiempo real a tu endpoint cuando ocurren acciones relevantes. El evento más crítico es transaction.pending — debes procesar y responder en menos de 2 segundos.

🔐
Verifica la autenticidad de cada webhook usando el header X-PagoASAP-Signature. Contiene un HMAC-SHA256 del payload firmado con tu webhook secret.

Eventos disponibles

⚡
transaction.pending
Transacción esperando aprobación. Responde con approve/decline en ≤2s.
✅
transaction.approved
Confirmación de transacción aprobada y procesada exitosamente.
❌
transaction.declined
Transacción rechazada por el partner, por reglas o por timeout.
🔄
transaction.reversed
Reversión o chargeback de una transacción previamente aprobada.
💳
card.created
Tarjeta virtual creada y lista para usar.
📦
card.delivered
Tarjeta física entregada al titular en la dirección registrada.
⚠️
balance.low
Tu balance prefunded está por debajo del umbral configurado.
💰
payout.completed
Payout procesado exitosamente hacia la cuenta destino.

Estructura de payload

{ "id": "evt_3Kp8nXmQrTs7", "object": "event", "type": "transaction.pending", "created_at": "2026-04-08T16:42:00.123Z", "data": { "id": "tx_8Yw3nBvRkP2m", "amount": 45.00, "currency": "USD", "card_id": "card_9Xp3nKmRtL8q", "cardholder_id": "ch_7fK2mNpQrXs9", "merchant": { "name": "Supermercado Central", "mcc": "5411", "city": "Caracas", "country": "VE" }, "decision_deadline": "2026-04-08T16:42:02.123Z" } }

API de Payouts

Envía dinero a cuentas bancarias, wallets digitales y beneficiarios en Venezuela y el exterior, utilizando la red de pagos de ASAP.

POST /payouts
Crear un payout hacia una cuenta o wallet
›
Body Parameters
amountnumberREQUIRED
Monto a enviar en la moneda especificada.
currencystringREQUIRED
Moneda del payout: USD, VES, EUR

Cuenta destino. Incluye type (bank_account, wallet, card) y datos del beneficiario.

Descripción del pago para el destinatario.

Clave única para evitar duplicados. Usa UUID v4.

Errores

Códigos de Error

PagoASAP usa códigos HTTP estándar con detalles estructurados en el body:

HTTPCódigoDescripción
400invalid_requestParámetros inválidos o faltantes.
401unauthorizedToken inválido, expirado o sin permisos.
402insufficient_prefunded_balanceBalance prefunded insuficiente para aprobar la TX.
404resource_not_foundEl recurso solicitado no existe.
409duplicate_requestIdempotency key ya fue procesada.
422kyc_requiredEl titular requiere completar KYC antes de emitir tarjeta.
429rate_limit_exceededDemasiadas requests. Límite: 100 req/min.
408tx_decision_timeoutNo respondiste approve/decline en 2 segundos.
500internal_errorError interno. Contacta support@pagoasap.com
